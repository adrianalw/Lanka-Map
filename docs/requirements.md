# Lanka Map — Requirements Document

## 1. Overview

Lanka Map is an interactive web application for exploring tourist locations across Sri Lanka. It provides a map-based interface for travellers to discover beaches, temples, wildlife reserves, waterfalls, hiking trails, historical sites, viewpoints, museums, and gardens.

---

## 2. Functional Requirements

### 2.1 Public Map Interface

| ID | Requirement | Priority |
|----|-------------|----------|
| FR-01 | Display an interactive map centred on Sri Lanka | Must Have |
| FR-02 | Show all location markers on the map | Must Have |
| FR-03 | Cluster nearby markers at lower zoom levels | Must Have |
| FR-04 | Filter locations by category (Beach, Temple, Wildlife, Hiking, Waterfall, Historical, Viewpoint, Museum, Garden, Other) | Must Have |
| FR-05 | Display a detail panel when a marker is clicked, showing name, description, entry fee, hours, and photo | Must Have |
| FR-06 | Provide a Google Maps directions link from each location detail panel | Must Have |
| FR-07 | Show the total count of visible locations | Should Have |
| FR-08 | Support horizontal scroll through category filters on mobile | Must Have |

### 2.2 Admin Panel

| ID | Requirement | Priority |
|----|-------------|----------|
| FR-09 | Restrict admin access behind a password-protected login page | Must Have |
| FR-10 | Persist admin session via a secure httpOnly cookie (7-day expiry) | Must Have |
| FR-11 | Allow admin to add new locations with name, category, coordinates, description, entry fee, hours, and photo | Must Have |
| FR-12 | Allow admin to edit existing locations | Must Have |
| FR-13 | Allow admin to delete locations | Must Have |
| FR-14 | Allow admin to upload photos to Supabase Storage | Must Have |
| FR-15 | Provide a search/filter in the dashboard location list | Should Have |

### 2.3 Contact Page

| ID | Requirement | Priority |

| FR-17 | Contact form must collect: name, email address, subject, and message | Must Have |
| FR-18 | Form submission must send an email to aalwishewa@gmail.com | Must Have |
| FR-19 | Show success/error feedback after form submission | Must Have |

### 2.4 How-To Guide

| ID | Requirement | Priority |
|----|-------------|----------|
| FR-20 | Provide a How-To guide page accessible from the main navigation | Must Have |

## 3. Non-Functional Requirements

| ID | Requirement | Detail |
|----|-------------|--------|
| NFR-01 | Performance | Initial page load under 3 seconds on a standard 4G connection |
| NFR-02 | Responsiveness | Fully usable on mobile (320px+), tablet, and desktop |
| NFR-03 | Accessibility | Navigation and form elements must have accessible labels |
| NFR-04 | Security | Admin password stored as environment variable; session cookie is httpOnly and Secure |
| NFR-05 | SEO | Page titles and meta descriptions set for all public pages |
| NFR-06 | Availability | Target 99.9% uptime via Vercel deployment |
| NFR-07 | Data Integrity | All location CRUD operations validated server-side before database write |

---

## 4. Technical Requirements

| Area | Requirement |
|------|-------------|
| Framework | Next.js 14 (App Router) with TypeScript |
| Map Library | Leaflet + react-leaflet with MarkerCluster |
| Tile Provider | OpenStreetMap (no API key required) |
| Database | Supabase (PostgreSQL) |
| Auth | Session-based (httpOnly cookie); no third-party OAuth required |
| File Storage | Supabase Storage for location photos |
| Email | Nodemailer via SMTP (Gmail) for contact form |
| Hosting | Vercel (recommended) |
| Styling | Tailwind CSS |

## 5. Data Requirements

### 5.1 Location Data Fields

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| id | UUID | Yes | Auto-generated by Supabase |
| name | string | Yes | Display name of the location |
| category | enum | Yes | One of 10 defined categories |
| lat | float | Yes | Latitude (WGS84) |
| lng | float | Yes | Longitude (WGS84) |
| description | string | Yes | Summary description (max ~500 chars) |
| entry_fee | string | Yes | Human-readable fee string (e.g. "Free", "LKR 1500") |
| hours | string | Yes | Human-readable opening hours |
| photo_url | string | No | Public URL from Supabase Storage |
| created_at | timestamp | Yes | Auto-generated |

### 5.2 Seed Data

- 157 locations pre-seeded via `scripts/seed.ts`
- Run with: `npm run seed`

---

## 6. Out of Scope (v1)

- User accounts / authentication for public visitors
- User-submitted reviews or ratings
- Real-time weather integration
- Multi-language support (Sinhala / Tamil)
- Offline / PWA support
- Paid booking or reservation system

---

## 7. Constraints

- Map tiles provided by OpenStreetMap (subject to OpenStreetMap usage policy)
- Supabase free tier limits: 500MB database, 1GB storage, 2GB bandwidth
- Contact form requires valid SMTP credentials in environment variables
- Admin password must be set via `ADMIN_PASSWORD` environment variable
